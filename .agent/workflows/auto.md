# Auto Commit & Push

Stage, auto-generate a commit message from diff, then commit and push to GitHub.

## Steps

// turbo-all

1. Stage all changes:
```
git add -A
```

2. Check if there are staged changes:
- Run `git diff --staged --quiet`
- If exit code is `0`, stop and report "No changes to commit"

3. Read staged diff and generate commit message automatically:
- Inspect staged changes with `git diff --staged`
- Write a concise commit message (Thai or English) that reflects the actual change
- Prefer repository style prefixes like `feat:`, `fix:`, `chore:`, `refactor:`, `docs:`

4. Commit using the generated message:
```
git commit -m "<auto-generated message>"
```

5. Push to current branch:
```
git push origin HEAD
```

## Notes
- Do not ask user for commit message unless the diff is ambiguous
- Commit message should focus on why and impact, not a raw file list
- If nothing changed, skip commit and push
- If push is rejected, run `git pull --rebase origin main` and push again
